    Ooyala-Video Cloud取り込みアダプタ:Backlot API を使用した取り込み

    このトピックでは、Backlot APIを使用して Video Cloud へのコンテンツの取り込みについて説明します。

    注:このトピックは、Video Cloud に移行した Ooyala カスタマーにのみ適用されます。

    はじめに

    Backlot APIを使用して、プログラムで Video Cloud にコンテンツをアップロードします。このアプローチにより、コンテンツ管理システム（CMS）またはワークフローを Video Cloud プラットフォームに直接統合できます。

    統合またはアプリは、いつでもCMSAPIおよびDynamicIngestAPIに移行できることに注意してください。

    を使用して、ビデオ、Backlot APIサムネイル、クローズドキャプションファイルをプログラムでアップロードおよび管理できます。

    Backlot API を使用してコンテンツを取り込む場合は、以下を参照してください。

    • ビデオおよびオーディオアセット（ビデオとサムネール用）
    • クローズドキャプションの操作

    ファイルのアップロードに使用される主な呼び出し（「資産」と呼ばれます）/v2/assetsがルートです。

    注:ヌルexternal_id、つまり値のない ( "" ) または値がのビデオを追加しないでください"null"。external_idこのようなヌル外部 ID は、後で検索することはできません。

    ビデオおよびオーディオアセット

    Ooyala に動画をアップロードすると、 Ooyala はアカウントに指定された設定を使用して自動的に変換し、プレーヤーを通じて動画を再生できるようにします。

    オーディオアセットは、ビデオトラックのないビデオとして Video Cloud に保存されます。

    H.264 で Ooyala に動画をアップロードすることをお勧めしますが、一般的に使用される多くのコーデックもサポートしています。

    新しいビデオアセットまたはオーディオアセットを作成するには:
    1. 基本プロパティを使用してアセットを作成する
    2. システムから「アップロード URL」を取得する
    3. 「アップロードURL」ごとに、適切なファイルチャンクをシステムに置きます
      注:各チャンクをアップロードするときは、他のリクエストと同様に、リクエストに署名しないでください。チャンクの内容を正確な URL に配置するだけです。これらの URL は既に署名されています。

    新しく作成したアセットのアップロードURLを取得する

        [GET] /v2/資産/ {asset_id} /uploading_urls
    注:各チャンクをアップロードするときは、他のリクエストと同様に、リクエストに署名しないでください。チャンクの内容を正確な URL に配置するだけです。これらの URL は既に署名されています。

    アップロードステータスの設定

        [PUT] /v2/資産/ {asset_id} /upload_status
    {
       「ステータス」: "アップロード済み」
    }

    ビデオのストリームを表示する

    この文脈におけるストリームという用語は、ビデオをトランスコードするためにシステムによって開始される内部からOoyalaへのプロセスを意味します。

    注:これらのストリームはシステムによって制御されるため、これらのストリームは制御または操作できません。名前の表示のみが可能で、変更することはできません。
        [GET] /v2/資産/ {asset_id} /ストリーム

    ビデオのソースファイルを表示する

        [GET] /v2/資産/ {asset_id} /source_file_info

    自動生成されたプレビュー画像の URL を取得する

        [GET] /v2/資産/ {asset_id} /generated_preview_images

    カスタムプレビュー画像をアップロード

        [POST] /v2/資産/ {asset_id} /preview_image_files
    <file_contents >

    プライマリプレビューイメージの設定を設定

    アセットのプライマリプレビューイメージのタイプを次のいずれかに設定します。
    • generated : 自動生成されたプレビュー画像を使用します
    • uploaded_file : アップロードされたカスタムプレビュー画像を使用します
    • remote_url :プレビュー画像の URL
        [置く] /v2/資産/ {asset_id} /primary_preview_image
    {
       「タイプ」: "生成された」|「uploaded_file」|「remote_url」
    }

    メタデータについて

    アセットには、最大 100 個のカスタムメタデータの名前と値のペアを設定できます。名前は 128 文字に制限され、その値は 2,048 文字に制限されます。

    例

    この例では、ビデオを作成します。

       「名前」:「ハニーアナグマ」,
       「file_name」:」honeybadger.avi「,
       「asset_type」: "ビデオ」,
       「file_size」: 398422、
       「chunk_size」: 100000
    }

    Video Cloud は、次のような応答を返します。

       「preview_image_url」: null,

       「説明」: null、
       「ステータス」: "アップロード中」
    }

    クローズドキャプションの操作

    Backlot API を使用して、動画やリモートアセット動画にクローズドキャプションを追加できます。

    キャプションでサポートされる形式

    サポートされている形式
    拡張子 説明
    .vtt ( WebVTT/) - this is the format required for HTML5 video players WebVTT は、HLS や DASH などの HTTP ストリーミング形式の埋め込みキャプションとサイドカーキャプションの両方に使用できます。
    .TTML ( 以前DXFP )-WebVTT に変換されます TTML (時限テキストマークアップ言語ファイル。通常、このタイプのクローズドキャプションファイルは、.dfxp (配布形式交換プロファイル) 拡張子を持つことになります。

    クローズドキャプションのアップロード

        [入れて] /v2/資産/ {asset_id} /closed_captions/ {dfxp_file_content}

    クローズドキャプションを取得する

        [GET] /v2/資産/ {asset_id} /closed_captions

    クローズドキャプションの更新

        [入れて] /v2/資産/ {asset_id} /closed_captions

    クローズドキャプションの削除

        [削除] /v2/資産/ {asset_id} /closed_captions
