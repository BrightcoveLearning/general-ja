    Ooyala-Video Cloud取り込みアダプタ:XML マニフェストファイル

    引き続き XML マニフェストで FTP、Aspera、または MRSS を使用して Video Cloud に動画を取り込むことができます。このトピックでは、XML の構造について説明します。
    注:このトピックは、Video Cloud に移行した Ooyala カスタマーにのみ適用されます。

    はじめに

    ビデオを取り込むときは、XML マニフェストファイルを使用して、追加のメタデータ（動画のタイトル、説明、フライト時間、ラベル、カスタムメタデータを含む）、およびサムネイルファイルとクローズドキャプションファイルを指定できます。FTP、Aspera、または MRSS を使用してコンテンツを取り込む場合は、XML マニフェストファイルを使用できます。

    注:このトピックは、OIS v2.5 に適用されます。

    メタデータの処理

    metadataビデオを取り込むときに含めると、次の 2 つの方法のいずれかで Video Cloud 動画フィールドにマップされます。

    1. デフォルトでは、各metadataキー/値のペアは、タグとしてmeta::key::valueの形式で追加されます。たとえば、<ooyala:metadata name="category">cooking</ooyala:metadata>はタグを生成しますmeta::category::cooking
    2. custom_fieldsを使用してメタデータを保存する場合は、ブライトコーブサポートがアカウントの設定を変更して変更することができます。次の点に注意してください。
      • メタデータキーに対応する名前を持つカスタムフィールドを作成する必要があります。
      • カスタムフィールドの数は既定で 10 に制限されています。そのため、メタデータを格納するためにそれ以上のカスタムフィールドが必要な場合は、サポートから追加のカスタムフィールドを要求する必要があります。
      • 各メタデータ項目に対応するカスタム項目がアカウントに存在しない場合、メタデータ操作全体が失敗します。

    XML マニフェスト要素

    XML マニフェストファイルでは、次の要素を使用できます。

    XML マニフェスト要素
    XML タグ 説明 必須/オプション
    <channel> 1 <item>つまたは複数の要素が含まれています。 必須
    <item> 取り込む単一のビデオを表します。XML マニフェストファイルでは、複数の動画（<item>タグ）を指定できます。このテーブルの残りのタグが含まれます。 必須
    <media:content> 取り込むビデオを指定します。次のいずれかの操作を行います。
    • アップロードした動画のファイル名。
    • MRSS を介した取り込み（「リモートホストされた MRSS フィードからの取り込み」を参照）では、ビデオが配置されている URL。URL には、http://またはを含めることができますhttps://。
    リモートアセットの場合は、<ooyala:remoteasset>代わりにを使用します。    		 必須（リモートアセットを除く）
    <media:title> ビデオの名前またはタイトル。 オプショナル
    <media:thumbnail> サムネイル (プレビューイメージ) を指定します。次のいずれかの操作を行います。
    • アップロードしたサムネイルのファイル名。
    • サムネイルが配置されている URL。URL には、http://またはを含めることができますhttps://。
    		 オプショナル
    <media:subTitle> クローズドキャプションファイルを指定します（DFXP、WebVTT、および SRT ファイルでサポートされます）。クローズドキャプションファイルの取り込みを参照してください。属性:
    • href ( 必須)-次のいずれかを指定します。
      • アップロードしたクローズドキャプションファイルのファイル名。例: caption-1.dfxp
      • クローズドキャプションファイルが格納されている URL。URL には、http://またはを含めることができますhttps://。
      lang ( オプション)-クローズドキャプションファイルの言語。2文字のコード。サポートされているクローズドキャプションを参照してください。
    • frameRate（オプション）-クローズドキャプションファイルのフレームレート。単位はフレーム/秒 (FPS) です。デフォルトは 30 です。古いクローズドキャプションファイルは 24 である可能性があります。
    注:指定した場合、langframeRateおよびに有効な値を入力するのはユーザーの責任です。有効な値は DFXP ファイルに追加されます。非準拠の値は無視されます。どちらの方法でも、クローズドキャプションファイルが Backlot に追加され、アセットにリンクされます。
    		 オプショナル
    <media:description> ビデオの説明。 オプショナル
    <link> ビデオを埋め込む永続的な URL。hosted_at作成したアセットのプロパティにマップします。 オプショナル
    <dcterms:valid> アセットを再生できる開始時間と終了時間を表すフライト時間。例: 
      <dcterms: > 有効な開始=2011-06-28T 15:15:00-07:00;
                 end=2022-12-01T 16:00:00-07:00; scheme=w3C-DTF
  </dcterms: 有効 >
    		 オプショナル
    <ooyala:labels> カンマで区切られた 1 つまたは複数のラベル。例: /sports,/sports/jogging。背景については、「ラベル」を参照してください。 オプショナル
    <media:keywords> ラベルを定義する別の方法（スラッシュなし）。コンマで区切られたコンテンツは、動画にタグとして追加されます。新しいタグは、同じ名前のタグがまだ存在しない場合にのみ作成されます。

    たとえば、<media:keywords>label2次のタグの場合、label1ラベルとが作成され、タグとしてビデオに追加されます。parentlabelという名前のラベルも作成され、childlabelその内部にラベルがネストされています。

      <メディア:キーワード > label1、label2、
      親ラベル/子ラベル < /メディア:キーワード >
    注:<media:keywords>タグがXMLマニフェストファイルまたはMRSSに含まれている場合、<ooyala:labels>の内容は無視されます。
    		 オプショナル
    <ooyala:embedcode> Video Cloud動画 ID。 オプション（新しい動画を追加するのではなく、動画を更新する場合は必須）
    <ooyala:profileguid> 処理プロファイルの ID。トランスコーディング中にマスタービデオを変換する必要がある形式を定義するエンコードのグループです。詳細については、Ooyala サポート担当者にお問い合わせください。 オプショナル
    <ooyala:metadata> ビデオのカスタムメタデータ。メタデータの種類ごとに 1 つの要素を設定できます。metadataが特別なタグとして動画に追加される方法の詳細については、メタデータの取り扱いセクションを参照してください。例: <ooyala:metadata name="category">cooking</ooyala:metadata> オプショナル
    <ooyala:remoteasset> 取り込むリモートアセットを指定します。リモートアセットとは、Backlot 以外の場所でホストされている動画です。次の属性を指定します。
    • durationInMs : メディア継続時間 (ミリ秒)
    • url :リモートアセットの URL。URL には、http://またはを含めることができますhttps://。
    この設定は、hosted_atリモートアセットのプロパティの値にマップされます。例: <ooyala:remoteasset durationInMs='5000' url="http://your_website/your_remote_assets.mp4" />
    注:リモートアセットが作成された後、さまざまな CDN への伝播が 60 秒以上遅れることがあります。リモートアセットの作成後にあまりにも早く要求すると、結果は CDN によってキャッシュされ、クリアには数分かかる場合があります。ベストプラクティス：作成後 30 秒または 60 秒待ち、 Backlot API [GET] /v2/assets/asset_idルートでクエリを実行し、リモートアセットの埋め込みコード（コンテンツ ID またはアセット ID）を取得した後、アセットの埋め込みに進みます。
    		 必須（リモートアセットのみ）
    <ooyala:profileguid> ビデオ処理に使用する Video Cloud インジェストプロファイルを指定します。アカウントのデフォルトプロファイルを使用しない場合は、代わりに使用するプロファイルを指定する必要があります。「取り込みプロファイルの管理」を参照してください。 オプショナル

    ステップバイステップ

    XML マニフェストファイルの作成

    XMLメタデータ・ファイルを作成するには、次の手順に従います。

    1. 動画やその他のコンテンツ（サムネイル、クローズドキャプションファイルなど）をアップロードします。
    2. XML を検証するテキストエディタ (できれば) を開きます。
    3. 必要な XML ヘッダーを追加します。例は次のとおりです。 
         <？xml version= "1.0" エンコード=" utf-8"？>
  <rss version= "2.0" xmlns: media=」http://search.yahoo.com/mrss/」
       xmlns: dcterms=」http://purl.org/dc/terms/」
       xmlns: fh=」http://purl.org/syndication/history/1.0」
       xmlns: ooyala=」http://www.ooyala.com/mrss/ ">
    4. channelコンテナを作成します。
    5. item上記の表の目的の要素を含む各ビデオのエントリを作成します。

      次の例では、XML は 2 つのビデオのメタデータを指定します。最初の動画では、ファイルとサムネイルはアップロードサーバー上に配置され、そのサーバーから取り出されます。2 番目のビデオでは、ファイルとサムネイルは既に Ooyala サーバーにアップロードされています。2番目のビデオには飛行時間もあります。

         <？xml version= "1.0" エンコード=" utf-8"？>
  <rss version= "2.0" xmlns: media=」http://search.yahoo.com/mrss/」
       xmlns: dcterms=」http://purl.org/dc/terms/」
       xmlns: fh=」http://purl.org/syndication/history/1.0」
       xmlns: ooyala=」http://www.ooyala.com/mrss/ ">
  
       <チャンネル >
             <アイテム >
              <メディア:content url=」http://mysite.com/upload/lacrosse_70.mov "/>
              <メディア:サムネール=」http://mysite.com/upload/lacrosse_70_previewimage.jpg "/>
              <メディア:> タイトル本当に存在するスポーツ:<ラクロス/メディア:タイトル >
              <メディア:> 説明私のビデオの説明 < /メディア:説明 >
              <ooyala: > ラベル/スポーツ/ラクロス、< /ホビーズ/ラクロス/Ooyala:ラベル >
              <リンク > http://mysite.com/videos/sports/real_sports.html < /link >
              <おいやな:メタデータ名= "Video_ID」> 70 < /ooyala: メタデータ >
              <おいやな:メタデータ名= "season_number」> 1 < /おいやな:メタデータ >
             </アイテム >
  
             <アイテム >
              <メディア:コンテンツ URL =」curling_71.mov "/>
              <メディア:サムネール=」curling_71_previewimage.jpg「/>
              <メディア:> タイトル本当に存在するスポーツ:<カーリング/メディア:タイトル >
              <メディア:> 説明私のビデオの説明 < /メディア:説明 >
              <ooyala: > ラベル/スポーツ/カーリング、< /ホビーズ/カーリング/ooyala: ラベル >
              <リンク > http://mysite.com/videos/sports/real_sports.html < /link >
              <Ooyala:メタデータ名= "Video_ID」> 71 < /おいやな:メタデータ >
              <おいやな:メタデータ名= "season_number」> 1 < /おいやな:メタデータ >
              <!— フライト時間を設定する — >
              <dcterms: > 有効な開始=2011-06-28T 15:15:00-07:00;
                  end=2022-12-01T 16:00:00-07:00; scheme=w3C-DTF
                  </dcterms: 有効 >
  
             </アイテム >
       </チャンネル >
  </rss >
    6. ファイルを XML として保存します。必ず一意の名前を指定してください。同じ名前の 2 つのファイルを指定し、もう一方のファイルを処理中にアップロードすると、2 つ目は無視されます。
    7. 選択したアップロード方法を使用して、ビデオのアップロード時にXMLマニフェストファイルをアップロードします。FTP または Aspera のいずれかを参照してください。
