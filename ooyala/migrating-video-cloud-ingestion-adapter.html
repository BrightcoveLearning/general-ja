新しいVideo Cloudオプション

Video Cloud には、動画の取り込みに標準的な 3 つのオプションがあります。

  • Studio のアップロードモジュール
  • 動的インジェスト API を使用したプルベースの取り込み
  • 動的取り込み API を使用したソースファイルのアップロード

またはウーヤラの摂取方法を使う

Video Cloud のお客様としては、上記のいずれかのオプションを使用できるが、ブライトコーブは Ooyala 取り込みシステム用のアダプターも構築しており、動画をビデオクラウドの取り込みシステムに誘導するため、 Ooyala Ingestion Service (OIS)を引き続き使用して以下を使用して Video Cloud に動画を送信

  • FTPS
  • アスペラ
  • MRSS フィードをリモートでホストする
  • バックロット API

既存のマニフェストオプションはすべて引き続き機能します。Ooyala の認証資格情報を引き続き使用すると、アダプターはこれらを必要な Video Cloud API の認証情報に自動的に変換します。

API ベースの取り込み
取り込みワークフローに、取り込み API を介した新しいコンテンツの直接取り込みが含まれる場合：クライアントトラフィックが Video Cloud に転送されるときに、既存の Ooyala API キーとシークレットを使用して、現在の Ooyala API エンドポイントへの API 呼び出しを行います。
FTP/アスペラベースの摂取
Video Coud「アップロードのみ」のユーザーアカウントのパスワードを Backlot で使用していたものとは異なるものにリセットしてください。Backlot の「アップロードのみ」アカウントで動画とメタデータの両方をサポートしている場合は、Video Cloud の「アップロードのみ」アカウントもそれを行います。同じ FTP/ASPERA エンドポイントに引き続き接続し、新しい Video Cloud の「アップロードのみ」ユーザーアカウント認証情報を使用してファイルをアップロードできます。アップロードされたファイルがアップロードされ、アップロードフォルダから削除されたら、Video Cloud にログインし、動画が追加されていることを確認します。動画と一緒に XML または CSV メタデータファイルをアップロードし、特定の取り込みプロファイルをターゲットにする場合は、profileguid 要素の下の XML または CSV フィールドに Video Cloud プロファイル名を指定する必要があります。「取り込みプロファイルの管理」を参照してください。
MRSS ベースの摂取
MRSS フィードをアカウントチームと共有してください。そうすれば、MRSS フィードを有効にできます。MRSS フィードのどの動画が以前に Backlot に取り込まれたかを検証できないため、MRSS フィードでは新しい動画のみが使用可能であることを確認します。古い動画がフィードにある場合、それらは取り込まれてしまい、Video Cloud に重複したコンテンツになります。MRSS フィードを配置したら、Video Cloud にログインして、コンテンツが取り込まれていることを検証できます。
ブライトコーブ動画クラウドスタジオを介した取り込み
Backlot との重要な違いは、コンテンツを取り込むために選択する取り込みプロファイルです。要件に最も適合する Video Cloud で取り込みプロファイルを選択してください。Video Cloud の取り込みプロファイルについては、「取り込みプロファイルの管理」を参照してください。。
