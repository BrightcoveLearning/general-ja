ページコンテンツ

    Ooyalaプレーヤーを使用してVideo Cloudコンテンツを配信する

    このトピックでは、動画が Video Cloud に移行された後も Ooyala ウェブプレーヤーと SDK プレーヤーが引き続き機能し、Brightcove プレーヤーへの移行が完了してもオンライン動画が機能し続ける方法について説明します。

    注:このトピックは、Video Cloud に移行した Ooyala カスタマーにのみ適用されます。

    はじめに

    ブライトコーブは、高度にカスタマイズ可能なウェブプレーヤーと Android、iOS、および tvOS 用のネイティブプレーヤーを提供しています。

    ただし、プレーヤーの埋め込みコードの置き換えとデバイスアプリの再構築には時間がかかることがわかっています。そのため、ブライトコーブでは、既存の Ooyala プレーヤーが Backloot ではなく Video Cloud から動画データを取得して作業を続行できるアダプタを構築しています。

    再生アダプタをテストするには、CXM と協力して、コンピュータおよびデバイスの DNS 設定を調整する必要があります。詳細については、「ブライトコーブ DNS サーバの追加」を参照してください。

    仕組み

    すべてのブライトコーブプレイヤーは、再生 API を使用して動画とプレイリストのデータを取得します。ただし、API リクエストを手動で設定する必要はありません。Web プレーヤーには、要求を処理するカタログライブラリがあり、デバイス SDK には、そのための組み込みメソッドがあります。

    Ooyala プレーヤーの機能を維持するために、ブライトコーブでは、動画と再生リストデータのプレーヤーリクエストを Player API と Video Cloud 内の同じ動画を指すアダプタを構築しました。

    注意:Ooyala ウェブプレーヤーの外観を変更することはできなくなります。現在のルックアンドフィールは保持されます。

    アナリティクス

    今後は、Video Cloud Analytics を使用します。既存の Ooyala プレーヤーを Video Cloud 動画で許可する修正には、Video Cloud 分析システムへの分析データの送信が含まれます。あなたのページやプレイヤーに加える必要のある変更はありません。

    Video Cloud 分析データには、Analytics モジュールの標準レポートとカスタムレポートまたは Analytics API のいずれかを使用してアクセスできます。API 経由で分析データを取得する場合は、「アナリティクス REST API から Brightcove Analytics API への移行」を参照してください。

    うまくいかないかもしれないもの

    Ooyala プレーヤーに統合されたアドオンがある場合、それらは引き続き機能します。ただし、これらのアドオンが利用できないビデオデータに依存する場合は、正しく機能しません。詳細については、 Ooyala ビデオフィールドと再生リストフィールドと Video Cloud フィールドのマッピングを参照してください。

    さらに、Ooyala API へのリクエストを行うアドオンがある場合、多くの場合、これらのエンドポイントは動作しませんが、Backlot API一部のエンドポイントは引き続き動作します。検出 API は動作しません。

    分析データは収集され、ブライトコーブアナリティクスシステムに送信されます。

    次のステップ

    Brightcove は、Video Cloud への移行時に Ooyala プレーヤーの動画と再生リストを維持することに努めますが、その機能とカスタマイズ性を最大限に活用するために、ブライトコーブプレーヤーに移行する計画を立てる必要があります。
